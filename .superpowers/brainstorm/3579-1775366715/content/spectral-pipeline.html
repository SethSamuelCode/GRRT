Step 2: Ray-March Through Disk (the change)

At each ray-march step — current vs spectral
CURRENT (RGB)
At step s:
1. Get local T, ρ, redshift g
2. Compute blackbody RGB
→ integrate Planck × CIE over 380-780nm
3. Apply redshift to RGB
4. Accumulate into 3 channels
R += emission_R × (1 - opacity)
G += emission_G × (1 - opacity)
B += emission_B × (1 - opacity)
NEW (Spectral)
At step s:
1. Get local T, ρ, redshift g
2. For each frequency bin νk:
→ B(νk, T) = 2hν³/c² × 1/(ehν/kT - 1)
3. Redshift each bin: νobs = g × νemit
4. Accumulate into N channels
I[k] += emission[k] × (1 - opacity[k])
for k = 0, 1, 2, ... N-1

Step 3: Output — The Spectral Data Cube

FITS file structure: 3D array
■ Image plane (x, y pixels)
■ Frequency axis (user-defined bins)
■ Each pixel has a full spectrum

Step 4: Post-Processing (external tools)

What you do with the FITS cube
DS9 / astropy
Pick a slice → monochrome image at that frequency (like Option C)
False Color
Map 3 bands → RGB channels (like NASA composites, Option B)
Blender Linear
Integrate spectrum × CIE → linear RGB → EXR for Blender
PNG
Tone-map any of the above → sRGB PNG
